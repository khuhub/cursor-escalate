# Workstream 05: CLI, Examples, README, E2E

## Goal

Implement the user-facing CLI, smoke example project, README, and e2e harness. The CLI should be a thin orchestration layer over `@looper/core`, not a second implementation of loop behavior.

## Branch

`looper/cli-docs-e2e`, created from `looper/foundation`.

## Owned Paths

- `packages/cli/**`
- `examples/smoke-task/**`
- `README.md`
- CLI/e2e tests
- Minimal dependency edits in `packages/cli/package.json`

Avoid editing:

- `packages/core/src/loop.ts` except for small exported API adjustments required by CLI.
- `packages/core/src/cursor.ts`
- `packages/core/src/scorer.ts`
- `packages/api/**`

## Implementation Requirements

### CLI Commands

Implement commands from section 8 of the source spec:

```bash
cursor-looper "/goal <text>"
cursor-looper "/goal <text>" --cloud <url>
cursor-looper rerun <loop_id>
cursor-looper status <loop_id>
cursor-looper show <loop_id> [--iteration n]
cursor-looper cancel <loop_id>
cursor-looper ladder
```

Flags:

- `--max-iterations`
- `--per-tier-cap`
- `--ladder a,b,c`
- `--threshold`

### CLI Behavior

- Default runtime is local cwd.
- `--cloud <url>` switches to cloud runtime with `autoCreatePR: false` in core/Cursor config.
- Live output:
  - iteration start line with model and tier.
  - streamed tool call names dimmed if possible.
  - assistant text live if exposed by core callback.
  - score line with per-criterion pass/fail markers. Keep ASCII fallback safe.
  - escalation banner with reason.
  - final outcome and progress bar.
- `status` prints progress bar, tier/model, and last scores.
- `show` prints rubric, criterion table, and diff; with `--iteration`, print node detail.
- `cancel` marks the artifact cancelled through core/store behavior. If live cancellation support is not available across process boundaries, document the limitation clearly in code and README.
- `ladder` resolves and prints available model ladder through core Cursor wrapper.

### Example Project

Create `examples/smoke-task`:

- Tiny Node project.
- Vitest test script.
- One intentionally failing test.
- Minimal source file where Cursor can make the test pass.
- Keep it small enough for real e2e to run quickly.

### E2E

- Real SDK e2e is skipped unless `CURSOR_API_KEY` exists.
- Test command runs:
  - `cursor-looper "/goal make the failing test in examples/smoke-task pass"`
  - against `examples/smoke-task`
  - with ladder forced to a single cheap model.
- Assert:
  - loop reaches `passed`.
  - artifact has at least one iteration.
  - iteration diff is non-empty.
  - rubric was generated by a real agent call.
- Include mocked CLI tests for status/show/argument parsing so normal CI does not need Cursor credentials.

### README

Document:

- Node 22.
- `CURSOR_API_KEY` from `cursor.com/dashboard/integrations`.
- `.env` / env-file setup.
- All CLI commands and flags.
- All API routes.
- Vercel API env vars:
  - `BLOB_READ_WRITE_TOKEN`
  - `LOOPER_API_TOKEN`
- Escalation policy.
- How comments mutate rubrics.
- E2E smoke behavior and how to run it.
- Explicitly state there is no UI beyond API payloads and minimal loop list.
